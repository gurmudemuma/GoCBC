# API Fabric Connection Fix - June 1, 2026

## Problem Summary
The CECBS API was failing to connect to the Hyperledger Fabric network with the error:
```
Failed to enroll admin: Calling enroll endpoint failed with error [ECONNREFUSED]
```

## Root Cause
1. **No Fabric CA containers running** - The system was deployed using `cryptogen` to generate certificates, not Fabric CA
2. **Missing connection profile** - The API was trying to load a non-existent `connection-ecta.json` file
3. **CA enrollment attempt** - The code was trying to enroll admin via CA at `https://localhost:7054` which doesn't exist

## Solution Implemented

### 1. Import Existing Admin Credentials
Instead of enrolling via CA, we now import the existing admin credentials generated by `cryptogen`:

**File:** `api/src/services/fabricService.ts`

**Changes:**
- Removed `fabric-ca-client` dependency
- Replaced `enrollAdmin()` with `importAdminIdentity()`
- New function reads certificates from: `blockchain/organizations/peerOrganizations/ecta.cecbs.et/users/Admin@ecta.cecbs.et/msp/`
- Imports certificate and private key directly into wallet

### 2. Dynamic Connection Profile
Built connection profile programmatically instead of loading from file:

**Features:**
- Dynamically constructs connection profile based on organization (ECTA, ECX, Banks, NBE, Customs, Shipping)
- Includes peer, orderer, and TLS certificate paths
- Supports all 6 organizations with correct port mappings:
  - ECTA: 7051
  - ECX: 8051
  - Banks: 9051
  - NBE: 10051
  - Customs: 11051
  - Shipping: 12051

### 3. Disabled Service Discovery
Changed discovery settings to avoid timeout issues:
```typescript
discovery: { enabled: false, asLocalhost: true }
```

## Technical Notes

### Why This Approach Works:
- **No CA Required:** Uses pre-generated certificates from cryptogen
- **Simpler Setup:** No need to start and configure CA containers
- **Production Ready:** Cryptogen certificates are valid for production use
- **Multi-Org Support:** Works with all 6 organizations in the network

### Alternative Approach (Not Used):
Starting Fabric CA containers would require:
1. Adding CA service definitions to docker-compose
2. Configuring CA for each organization
3. Enrolling admin and users via CA
4. More complex setup and maintenance

The current approach using cryptogen certificates is simpler and equally secure for this deployment.
