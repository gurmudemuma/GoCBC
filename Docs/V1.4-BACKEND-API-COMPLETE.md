## New API Routes Added

### 1. Banking Routes (`/api/v1/banking`)

#### Letter of Credit Endpoints:
- `POST /banking/lc/request` - Request new LC
- `POST /banking/lc/approve` - Bank approves LC
- `POST /banking/lc/issue` - Bank issues LC
- `GET /banking/lc/:lcId` - Get LC details
- `PUT /banking/lc/:lcId/status` - Update LC status
- `GET /banking/lc/exporter/:exporterId` - Get LCs by exporter
- `GET /banking/lc/status/:status` - Get LCs by status

#### Payment & SWIFT Endpoints:
- `POST /banking/payment/initiate` - Initiate payment
- `POST /banking/payment/:paymentId/documents` - Submit documents
- `POST /banking/payment/:paymentId/verify` - Verify documents
- `POST /banking/payment/:paymentId/swift/initiate` - Initiate SWIFT transfer
- `POST /banking/payment/:paymentId/swift/confirm` - Confirm SWIFT receipt
- `POST /banking/payment/:paymentId/settle` - Settle payment with retention
- `GET /banking/payment/:paymentId` - Get payment details
- `GET /banking/payment/exporter/:exporterId` - Get payments by exporter
- `GET /banking/payment/swift/:swiftReference` - Find by SWIFT reference

## Authentication & Authorization

All protected routes require:
- Valid JWT token in `Authorization: Bearer {token}` header
- User must have appropriate organization role
- Chaincode functions check organization-specific permissions

---

## Error Handling

All endpoints return standardized error responses:
```json
{
  "success": false,
  "error": "Error message description"
}
```

Common HTTP status codes:
- `200` - Success
- `400` - Bad request (invalid parameters)
- `401` - Unauthorized (missing/invalid token)
- `403` - Forbidden (insufficient permissions)
- `404` - Resource not found
- `500` - Internal server error
