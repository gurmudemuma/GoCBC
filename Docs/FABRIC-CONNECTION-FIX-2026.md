# Fabric CA Connection Fix - June 1, 2026

## Problem Summary
The API was failing to connect to the Hyperledger Fabric network with the error:
```
Failed to enroll admin: Calling enroll endpoint failed with error [ECONNREFUSED]
```

## Root Cause
1. **No Fabric CA containers running** - The system was deployed using `cryptogen` to generate certificates, not Fabric CA
2. **Missing connection profile** - The `connection-ecta.json` file didn't exist
3. **Wrong approach** - Code was trying to enroll admin via CA (port 7054) which wasn't running

## Solution Implemented

### 1. Import Existing Admin Credentials
Instead of enrolling via CA, we now import the admin identity from the cryptogen-generated certificates:

**Location**: `api/src/services/fabricService.ts`

```typescript
private async importAdminIdentity(): Promise<void> {
  // Read certificate from: blockchain/organizations/peerOrganizations/ecta.cecbs.et/users/Admin@ecta.cecbs.et/msp/signcerts/
  // Read private key from: blockchain/organizations/peerOrganizations/ecta.cecbs.et/users/Admin@ecta.cecbs.et/msp/keystore/
  
  const x509Identity = {
    credentials: {
      certificate,
      privateKey,
    },
    mspId: 'ECTAMSP',
    type: 'X.509',
  };
  
  await this.wallet.put('admin', x509Identity);
}
```

### 2. Build Connection Profile Dynamically
Created a method to build the connection profile programmatically instead of reading from a file:

```typescript
private buildConnectionProfile(): any {
  return {
    name: 'cecbs-network',
    version: '1.0.0',
    channels: {
      coffeechannel: {
        orderers: ['orderer.cecbs.et'],
        peers: {
          'peer0.ecta.cecbs.et': {
            endorsingPeer: true,
            chaincodeQuery: true,
            ledgerQuery: true,
            eventSource: true,
          },
        },
      },
    },
    organizations: {
      ecta: {
        mspid: 'ECTAMSP',
        peers: ['peer0.ecta.cecbs.et'],
      },
    },
    peers: {
      'peer0.ecta.cecbs.et': {
        url: 'grpcs://localhost:7051',
        tlsCACerts: {
          path: '../blockchain/organizations/peerOrganizations/ecta.cecbs.et/peers/peer0.ecta.cecbs.et/tls/ca.crt',
        },
        grpcOptions: {
          'ssl-target-name-override': 'peer0.ecta.cecbs.et',
          'grpc.keepalive_time_ms': 120000,
        },
      },
    },
    orderers: {
      'orderer.cecbs.et': {
        url: 'grpcs://localhost:7050',
        tlsCACerts: {
          path: '../blockchain/organizations/ordererOrganizations/cecbs.et/orderers/orderer.cecbs.et/tls/ca.crt',
        },
        grpcOptions: {
          'ssl-target-name-override': 'orderer.cecbs.et',
        },
      },
    },
  };
}
```

### 3. Disable Service Discovery
Changed discovery settings to avoid timeout issues:

```typescript
const connectionOptions = {
  wallet: this.wallet,
  identity: 'admin',
  discovery: { enabled: false, asLocalhost: true }, // Disabled discovery
  eventHandlerOptions: {
    commitTimeout: 100,
  },
};
```

### 4. Removed Fabric CA Client Dependency
Removed the unused `fabric-ca-client` import since we're not using CA enrollment:

```typescript
// REMOVED: const FabricCAServices = require('fabric-ca-client');
```

### Why This Approach Works
1. **No CA Required**: Cryptogen already created all necessary certificates
2. **Simpler Setup**: No need to run and manage CA containers
3. **Production Ready**: Cryptogen certificates are valid for production use
4. **Faster Startup**: No enrollment process, just import existing credentials
